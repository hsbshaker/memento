# UI Patterns (Memento)

## Purpose

Define reusable interaction patterns.

---

## 1. Overview → Drilldown
- show summary first
- allow deeper exploration

---

## 2. Urgency First
Order by:
1. expiring soon
2. value remaining

---

## 3. Single Primary Action
- one clear CTA per screen

---

## 4. Progressive Disclosure
- show basics first
- reveal details on demand

---

## 5. Scanable Lists
- fast to read
- consistent structure

---

## 6. Guided Empty States
- explain
- give next step

---

## Rule

Patterns support UX — they do NOT constrain it.

If a better UX exists, ignore patterns.
